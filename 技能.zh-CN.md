# InkOS - 自主小说写作智能体

InkOS 是一个由 LLM 智能体驱动的自主小说创作 CLI 工具。它编排了一条由 5 个智能体组成的流水线（Radar → Architect → Writer → Auditor → Reviser），用于生成、审校和修订小说内容，并保持风格一致性与质量控制。

## 适用场景

- **小说写作**：创作和续写中文网文题材的小说或长篇作品
- **批量生成章节**：稳定生成多章内容并保持质量一致
- **风格模仿**：分析参考文本并采用其写作风格
- **衍生作品创作**：在保持母作设定一致的前提下创作前传、续作或外传
- **质量审校**：检测 AI 生成痕迹，并执行 32 维质量检查
- **题材探索**：分析趋势并创建自定义题材规则

## 初始设置

### 首次使用

```bash
# 初始化项目目录（会创建配置结构）
inkos init my-writing-project

# 配置你的 LLM 服务提供方（OpenAI、Anthropic 或兼容接口）
inkos config set-global --provider openai --base-url https://api.openai.com/v1 --api-key sk-xxx --model gpt-4o
```

### 查看系统状态

```bash
# 检查安装与配置
inkos doctor

# 查看当前配置
inkos status
```

## 常见工作流

### 工作流 1：创建一部新小说

1. **初始化并创建书籍**：

   ```bash
   inkos book create --title "My Novel Title" --genre xuanhuan --chapter-words 3000
   ```

   - 题材：`rebirth`（重生流）、`system`（系统流）、`onlinegame`（网游数据流）、`apocalypse`（末日生存流）、`interstellar`（星际流）、`xuanhuan`（玄幻）、`xianxia`（仙侠）、`urban`（都市）、`horror`（恐怖）、`other`（其他）
   - 返回后续所有操作都会用到的 `book-id`
2. **生成初始章节**（例如 5 章）：

   ```bash
   inkos write next book-id --count 5 --words 3000 --context "young protagonist discovering powers"
   ```

   - `write next` 命令会运行完整流水线：草稿 → 审校 → 修订
   - `--context` 为 Architect 和 Writer 智能体提供方向性提示
   - 返回包含章节详情与质量指标的 JSON
3. **审阅并批准章节**：

   ```bash
   inkos review list book-id
   inkos review approve-all book-id
   ```
4. **导出书籍**：

   ```bash
   inkos export book-id
   ```

### 工作流 2：续写已有小说

1. **列出你的书籍**：

   ```bash
   inkos book list
   ```
2. **从最新章节继续写**：

   ```bash
   inkos write next book-id --count 3 --words 2500 --context "protagonist faces critical choice"
   ```

   - InkOS 会维护 7 份事实文件（世界状态、角色矩阵、情绪弧线等）以保障一致性
   - 如果项目中只有一本书，可以省略 `book-id`，由系统自动识别
3. **审阅并批准**：

   ```bash
   inkos review approve-all
   ```

### 工作流 3：风格模仿

1. **分析参考文本**：

   ```bash
   inkos style analyze reference_text.txt
   ```

   - 分析词汇、句式结构、语气和节奏
2. **将风格导入你的书籍**：

   ```bash
   inkos style import reference_text.txt book-id --name "Author Name"
   ```

   - 后续所有章节都会采用该风格画像
   - 风格规则会成为 Reviser 审校标准的一部分

### 工作流 4：创作衍生作或前传

1. **导入母作设定**：

   ```bash
   inkos import canon spinoff-book-id --from parent-book-id
   ```

   - 会关联母作中的世界状态、角色和事件
   - Reviser 会强制校验设定一致性
2. **继续创作衍生作**：

   ```bash
   inkos write next spinoff-book-id --count 3 --context "alternate timeline after Chapter 20"
   ```

   - 生成器会遵守母作设定约束
   - 所有修订都会维持世界观与剧情一致性

### 工作流 5：细粒度控制（草稿 → 审校 → 修订）

如果你需要分别控制流水线的每个阶段：

1. **只生成草稿**：

   ```bash
   inkos draft book-id --words 3000 --context "protagonist escapes" --json
   ```
2. **审校章节**（32 维质量检查）：

   ```bash
   inkos audit book-id chapter-1 --json
   ```

   - 返回节奏、对白、描写、世界构建等指标
3. **按指定模式修订**：

   ```bash
   inkos revise book-id chapter-1 --mode polish --json
   ```

   - 模式：`polish`（润色）、`spot-fix`（定点修复）、`rewrite`（重写）、`rework`（结构重做）、`anti-detect`（降低 AI 痕迹）

### 工作流 6：监控平台趋势

```bash
inkos radar scan
```

- 分析热门题材、常见套路和读者偏好
- 为 Architect 提供新书策划建议

### 工作流 7：检测 AI 生成内容

```bash
# 检测指定章节中的 AIGC 痕迹
inkos detect book-id

# 深度扫描全部章节
inkos detect book-id --all
```

- 使用 11 条确定性规则（不消耗 LLM 成本）并可选配 LLM 校验
- 返回检测置信度和可疑片段

## 进阶：自然语言智能体模式

适用于更灵活、对话式的请求：

```bash
inkos agent "写一部都市题材的小说，主角是一个年轻律师，第一章三千字"
```

- 智能体会解析自然语言并调用合适的命令
- 适合复杂的多步骤请求

## 核心概念

### Book ID 自动识别

如果你的项目里只有一本书，大多数命令都可以将 `book-id` 设为可选项。为了简洁，可以省略：

```bash
# 显式指定
inkos write next book-123 --count 1

# 自动识别（仅当项目中只有一本书时）
inkos write next --count 1
```

### --json 标志

所有内容生成类命令都支持 `--json` 输出结构化结果，这对于程序化调用非常重要：

```bash
inkos draft book-id --words 3000 --context "guidance" --json
```

### 事实文件（长期记忆）

InkOS 会为每本书维护 7 份文件以保障连贯性：

- **世界状态**：地图、地点、科技水平、力量体系
- **角色矩阵**：姓名、关系、人物弧线、动机
- **资源账本**：世界内物品、金钱、战力等级
- **章节摘要**：事件、推进、伏笔
- **支线看板**：活跃与休眠中的支线、钩子
- **情绪弧线**：角色的情感发展轨迹
- **待回收钩子**：尚未回收的悬念与对读者的承诺

所有智能体都会引用这些文件，以维持长期一致性。

### 上下文提示

`--context` 参数会为 Writer 和 Architect 提供方向性提示：

```bash
inkos write next book-id --count 2 --context "protagonist discovers betrayal, must decide whether to trust mentor"
```

- `context` 是可选的，但强烈建议提供，以提升叙事连贯性
- 同时支持英文和中文

## 错误处理

### 常见问题

**"book-id not found"**

- 通过 `inkos book list` 确认 ID 是否正确
- 确保你位于正确的项目目录中

**"Provider not configured"**

- 使用有效凭据运行 `inkos config set-global`
- 通过 `inkos doctor` 检查 API Key 与 Base URL

**"Context invalid"**

- 确保 `--context` 的值是字符串（多词内容请加引号）5
- `context` 可以使用英文或中文

**"Audit failed"**

- 检查章节是否存在编码问题
- 确保 `chapter-words` 与实际字数匹配
- 尝试使用 `inkos revise --mode rewrite` 重新修订

### 运行守护进程模式

适用于长时间运行的任务：

```bash
# 启动后台守护进程
inkos up

# 停止后台守护进程
inkos down

# 守护进程会自动处理排队中的章节任务
```

## 最佳实践建议

1. **提供更丰富的上下文**：`--context` 提示越具体，叙事通常越连贯
2. **优先建立风格**：如果要模仿作者，先执行 `inkos style import`
3. **定期审阅**：使用 `inkos review` 尽早发现问题
4. **关注审校指标**：查看 `inkos audit` 结果，定位质量瓶颈
5. **合理使用衍生作机制**：写前传或续作前先导入 canon
6. **批量生成章节**：一次生成多章通常更利于连续性
7. **频繁导出**：通过 `inkos export` 定期保留备份
